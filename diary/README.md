# Project Diary

Purpose: this diary is compact project memory for AI continuation. It is not a transcript, changelog, task tracker, or replacement for `AGENTS.md`.

## Files
- `current.md` - short snapshot of where the project is now.
- `decisions.md` - durable decisions that should not be rediscovered.
- `days/YYYY-MM-DD.md` - compact daily notes.

## Relationship To Other Memory
- `AGENTS.md` - permanent project instructions: commands, conventions, constraints, workflow rules.
- `Summarizations.md` - session restart handoff.
- `diary/current.md` - compact current snapshot, not a plan.
- `diary/decisions.md` - decisions that should survive context resets.

## Write Protocol
- Write only information that helps continue the project after a context reset.
- Do not store transcripts, cosmetic-only notes, command noise, or temporary dead ends.
- Unfinished work belongs in the relevant daily note under `Carry Forward`, or in an ExecPlan if it needs a real plan.
